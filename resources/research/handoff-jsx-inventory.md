## Swap Risks

- `window.*` globals have no direct Svelte analog. The prototype relies on script order and `tryRender` polling (`index.html:35`-`index.html:58`, `index.html:176`-`index.html:184`), plus cross-file references such as `window.FocusGraphView` (`explorer.jsx:283`), `window.nodeOf` (`explorer.jsx:55`), and `window.KindBadge` (`graph-focus.jsx:61`). Porting should replace this with explicit module imports or a generated registry.
- React style numeric values will lose units if naively stringified. Examples: fixed artboard sizes (`home-editorial.jsx:66`, `doc-classic.jsx:52`), shared badge size (`shared.jsx:80`), and graph pill geometry (`graph-focus.jsx:47`). React adds `px` for many numeric CSS properties; Svelte style strings do not.
- Inline styles are too dynamic for a simple string copier. Dynamic ternaries and computed values drive active states, layout, and graph geometry (`explorer.jsx:198`-`explorer.jsx:203`, `graph-focus.jsx:49`-`graph-focus.jsx:58`, `doc-parts.jsx:99`-`doc-parts.jsx:109`), and 492 extracted style props reference CSS vars.
- Tailwind CDN output is not the same as the target Tailwind v4 build. The export depends on arbitrary utilities and variants such as `grid-cols-[1fr_minmax(480px,_44%)]` (`doc-split.jsx:69`), `hover:bg-[color:var(--panel-muted)]` (`doc-split.jsx:89`), and `animate-[fadeIn_.12s_ease]` (`graph-focus.jsx:88`). A Svelte/Tailwind v4 port needs static class discovery or safelisting for generated classes and arbitrary values.
- SVG conversion is not just tags. It includes camelCase SVG attributes (`strokeWidth`, `strokeLinecap`, `textAnchor`, `fillOpacity`) in shared icons and graphs (`shared.jsx:5`, `graph-focus.jsx:250`-`graph-focus.jsx:253`), computed path strings (`graph-focus.jsx:160`-`graph-focus.jsx:163`, `graph-banded.jsx:73`-`graph-banded.jsx:80`), gradients/patterns (`graph-focus.jsx:184`-`graph-focus.jsx:193`, `graph-constellation.jsx:85`-`graph-constellation.jsx:96`), and SVG/HTML layering (`graph-focus.jsx:182`-`graph-focus.jsx:183`, `graph-focus.jsx:289`-`graph-focus.jsx:312`).
- `dangerouslySetInnerHTML` must be handled intentionally. The only product use is bundled SVG icon markup (`doc-parts.jsx:213`-`doc-parts.jsx:216`), which can map to Svelte `{@html}` if the input remains trusted and local.
- React fragments with keys need special handling. `React.Fragment key={i}` is used for breadcrumbs (`doc-parts.jsx:188`, `explorer.jsx:221`), and Svelte's keyed each block should carry the key rather than a fragment wrapper.
- Babel/modern syntax is present. All scripts are `type="text/babel"` (`index.html:35`-`index.html:58`), and source uses optional chaining (`explorer.jsx:118`-`explorer.jsx:121`), nullish assignment (`graph-data.jsx:114`-`graph-data.jsx:115`, `graph-refined.jsx:98`), numeric separators (`home-library.jsx:9`-`home-library.jsx:19`), object spread (`graph-refined.jsx:98`), and JSX prop spreads (`doc-classic.jsx:93`, `shared.jsx:5`).
- The export uses inline SVG `Icon.*` helpers, not imported Lucide icons. Evidence: icon factories are in `shared.jsx` (`shared.jsx:4`-`shared.jsx:17`) and extra icon factories are added in `doc-parts.jsx` (`doc-parts.jsx:212`-`doc-parts.jsx:222`). A scan found no Material Symbols class usage; the only `symbol` match is prose in `doc-reading.jsx:4`. Since the target project already uses Svelte 5, Tailwind v4, and Lucide icons, reconciliation should map these prototype icons and `KindBadge`/`KindIcon` to target icon/token components rather than porting the global `Icon` object literally.
- Some hover styling mutates DOM element styles directly rather than state. Explorer tree and doc cards change `currentTarget.style.background` / `borderColor` in handlers (`explorer.jsx:63`-`explorer.jsx:64`, `explorer.jsx:133`-`explorer.jsx:134`). Svelte should convert these to class/style state or CSS hover rules.
- Design-tool chrome is much more React/DOM-heavy than product UI. It injects CSS (`design-canvas.jsx:30`-`design-canvas.jsx:107`), uses React context (`design-canvas.jsx:110`, `design-canvas.jsx:495`, `design-canvas.jsx:820`), portals (`design-canvas.jsx:873`-`design-canvas.jsx:944`), contentEditable (`design-canvas.jsx:803`-`design-canvas.jsx:811`), host `window.omelette` (`design-canvas.jsx:131`, `design-canvas.jsx:166`), and postMessage/localStorage controls (`tweaks-panel.jsx:171`, `tweaks-panel.jsx:216`, `tweaks-panel.jsx:220`). Do not include this in a product swap unless the canvas itself is being ported.
